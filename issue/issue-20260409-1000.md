# Issue: Optimasi Konfigurasi Runtime Web (Windows)

## Deskripsi
Meningkatkan efisiensi dan konsistensi konfigurasi runtime (Nginx, MariaDB, PostgreSQL) pada platform Windows menggunakan utilitas `sed.exe` kustom dan script batch yang terstandarisasi.

## Sub-Tasks

### 1. Pengembangan Tooling (`utils/executable_utils/`)
- [ ] **Optimasi `sed_utils.c`**: Menambahkan fitur multi-replace (misal: `--replace A B --replace C D`) agar dapat mengganti port dan root directory dalam satu pemanggilan.
- [ ] **Rebuild**: Membangun ulang `sed.exe` dengan fitur baru dan memastikannya tersedia di `utils/windows/bin/`.

### 2. Standarisasi Script Batch (`utils/windows/`)
- [ ] **Refactor `nginx-port.bat`**:
    - Menggunakan `sed.exe` baru.
    - Mengambil template dari `utils/baseconfig/windows/nginx.conf.template`.
    - Menerima 2 parameter: `%PORT%` dan `%ROOT%`.
- [ ] **Refactor `mariadb-port.bat`**:
    - Menggunakan `sed.exe`.
    - Mengambil template dari `utils/baseconfig/windows/mariadb.conf.template`.
- [ ] **Refactor `postgres-port.bat`**:
    - Sinkronisasi cara penanganan port (saat ini menulis ke `postgresql.auto.conf`).

### 3. Integrasi & Deployment (`resource/`)
- [ ] **Sinkronisasi Asset**: Menyalin script batch dan binary pendukung dari `utils/` ke folder `resource/` proyek Flutter.
- [ ] **Verifikasi Path**: Memastikan script dapat menemukan binary `sed.exe` relatif terhadap lokasi eksekusinya di dalam bundle Flutter.

### 4. Implementasi UI & Logika (`lib/components/part/settings.dart`)
- [ ] **Update `_applySettings`**:
    - Menyesuaikan `workingDirectory` untuk menunjuk ke folder resource yang benar (menggunakan `Platform.resolvedExecutable` atau path relatif).
    - Memperbaiki parameter yang dikirim ke `nginx-port.bat` (Port dan Root).
    - Memastikan status `onEdits` diperbarui dengan benar setelah penerapan.
